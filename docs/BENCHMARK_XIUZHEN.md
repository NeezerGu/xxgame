# BENCHMARK_XIUZHEN — 系统矩阵与补齐建议

> 目的：列出对标修仙类玩法时的能力差异，明确缺口与补齐路线，作为后续设计/实现的 source of truth。所有命名与数值均为原创且可替换主题。

| 对标系统 | 当前已有能力 | 缺口 | 建议实现（秘术工坊体系命名） | 优先级 & 依赖 |
| --- | --- | --- | --- | --- |
| 阶位线 / 主线阶段 | 现有 Essence/R/Insight 资源循环，升华软重置，Tab 结构（委托/升级/研究/升华/调试），确定性 tick + 离线结算基础。 | 缺少阶段化解锁与主线目标，无法引导玩家感知长线成长。 | 建立「阶位」主线：以阶段节点串联解锁（提升产出上限、开放额外委托类型），与升华形成双轨进度。阶位节点配置在 data 文件，可定义前置（Reputation/研究）与奖励（倍率、槽位）。 | 高优先级；依赖资源与委托基础，可先于其他系统落地，Roadmap M5。 |
| 炼金（养成消耗） | 现有研究点（R）与升级系统，可作为基础产出与消耗接口；存档/离线结算可支撑长时生产。 | 缺少配方、产线、库存，对应界面与排程逻辑。 | 设计「灵炉炼金」体系：引入配方（消耗 Essence/R/新资源）、生产时长、品质系数；队列化生产（可预设 3~5 个槽），完成后产出增益药剂（被动 Buff 或一次性消耗）。支持自动续杯研究。 | 中高优先级；依赖多资源与库存（Roadmap M6），UI 可沿用委托卡片风格。 |
| 锻造（装备打造） | 委托卡片与升级乘区可复用成「加工」骨架；已有研究树可提供前置。 | 无装备品类、部位、随机词缀/品质定义，缺少持久化装备位与效应。 | 定义「器胚打磨」：部位（器印/符核/护阵）各有基础词条，锻造时消耗资源 + 药剂增益，产出带随机前缀/后缀的器物。装备位与属性加成为数据驱动，词条生成使用存档内 seed 保证确定性。 | 中优先级；依赖炼金 Buff 与多资源；需新增装备存储区与鉴定流程，Roadmap M7。 |
| 装备系统（穿戴/强化） | 升级系统可提供线性/指数成本模版；UI 有升级 Tab 可展示数值。 | 缺少部位、耐久/强化、评分与搭配约束。 | 设计「符匣」界面：每个部位可穿戴一件器物，支持强化（消耗 Essence/专用材料）、修复。装备评分反哺 Reputation，决定高阶委托/阶位解锁。强化成功率与加成公式写入 data，保持确定性。 | 中优先级；依赖锻造产物；与阶位线/委托解锁耦合。 |
| 学徒（角色/分工） | 现有委托槽概念可转化为任务/派遣槽；研究树可解锁自动化。 | 无学徒实体、属性成长、分工收益；缺少任务/外勤入口。 | 引入「学徒卷轴」：生成若干学徒，拥有性格/专精（决策树固定，seed 决定），可派往炼金/锻造/外勤探索；行动结果影响产出与概率事件。学徒经验与好感驱动额外乘区或解锁。 | 中低优先级；依赖多系统接口（炼金/锻造/外勤探索），Roadmap M8。 |
| 外勤探索 / 冒险 | 现有离线模拟与 FAST_FORWARD 工具适合长时任务结算。 | 缺少地图/事件/奖励结构，无法产生自动化资源流或探索反馈。 | 构建「远野委托」：以简单节点图形式呈现，每个节点有时长、需求、风险与奖励；学徒派遣后按确定性表决算，支持离线推进。事件表与掉落表数据驱动，可受装备/药剂/阶位影响。 | 中低优先级；依赖学徒与装备，Roadmap M8。 |
| 神国 / 领域建造 | 当前有升华软重置与 Insight 资源，可作为长线循环。 | 缺少建筑格子、长期加成与跨周目目标。 | 设计「域构阵盘」：升华后可在阵盘上放置建筑节点（提供全局乘区、自动化、额外委托槽），消耗 Insight 与稀有材料。布局数据化，可迁移/扩展 schema。 | 低优先级；需在其他核心循环稳定后上线，可与升华迭代绑定。 |

### 补充说明
- 所有命名与具体数值均为占位，可换皮；实现时保持 deterministic tick、数据驱动与可迁移存档。
- 每个系统应提供最小可用的 UI 占位与调试入口，避免与现有 Tab 冲突。
