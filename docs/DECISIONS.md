# DECISIONS — 关键设计记录

1. **确定性优先的模拟循环**：所有 tick、离线结算与 RNG 基于存档 seed 与固定步长，避免浮点抖动；便于重放、调试与回归。
2. **契约驱动的节奏**：与常见被动产出不同，核心动力来自可选契约的约束与奖励，促使玩家在短周期内做选择，同时保留放置收益。
3. **研究分支代替硬直线科技**：采用分支/网状研究，提供槽位扩展、速度、自动化等差异化路径，防止单一路线导致线性化体验。
4. **Ascend 奖励与保留的克制设计**：Insight 收益为缓增（对数/根号），避免爆炸式滚雪球；保留研究与少量关键升级，保障重开动力而非重置惩罚。
5. **数据驱动与可迁移性**：数值、契约模板、研究树放在 `src/engine/data/*.ts`（或同层配置），配合 schemaVersion 与迁移函数，降低平衡与主题变更成本。
6. **离线收益有界且透明**：离线上限与结算规则固定且在 UI 明示，确保玩家预期稳定，不因后台挂机导致失衡。
7. **占位主题与原创命名**：使用“秘术工坊/契约放置”等可替换占位，不采用外部 IP、美术或数值表，确保后续换皮与合规。
